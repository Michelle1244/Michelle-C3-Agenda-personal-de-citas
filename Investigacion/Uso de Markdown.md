## **Qué es Markdown**

Markdown es un sistema de marcado simple que permite estructurar texto de manera clara sin necesidad de utilizar herramientas complejas. Su propósito es que el contenido escrito sea comprensible tanto en su forma original como cuando se transforma a otros formatos. Utiliza una sintaxis minimalista —como símbolos para encabezados, listas o énfasis— que facilita la creación de documentos ordenados y fáciles de interpretar.

Este lenguaje se ha vuelto muy común en entornos de desarrollo debido a que agiliza la elaboración de documentación técnica, notas y archivos README, ofreciendo una forma rápida y eficiente de presentar información sin depender de editores avanzados.



##### **Archivos README**

&nbsp;El archivo README es un componente esencial en cualquier proyecto, pues actúa como la carta de presentación para quienes lo visitan por primera vez. En él se ofrece una visión general del proyecto, explicando su propósito, las funciones principales y las instrucciones necesarias para instalarlo o ponerlo en marcha. Su objetivo es orientar rápidamente al usuario y facilitar la comprensión del funcionamiento del proyecto desde el inicio.



##### **Sistemas de Control de Versiones**

&nbsp;Plataformas como GitHub interpretan automáticamente los archivos con extensión .md, aplicando formato al contenido sin que el usuario tenga que hacer nada. Gracias a esto, los documentos escritos en Markdown se muestran con una apariencia ordenada, legible y visualmente estructurada, lo que facilita su lectura y comprensión dentro del repositorio.




## **Ventajas de usar Markdown con GitHub** 

Los documentos escritos en Markdown son fáciles de interpretar incluso en su forma básica, pero GitHub se encarga de darles formato automáticamente, mostrándolos como páginas limpias, ordenadas y agradables de leer. Además, redactar en Markdown resulta mucho más rápido que trabajar con código HTML u otros lenguajes de marcado más complejos, lo que hace que la creación de documentación sea mucho más eficiente.

#### **Mayor precisión en el control de cambios**

Al tratarse de archivos en texto plano, Git puede identificar cada modificación de manera detallada, permitiendo un seguimiento exacto de los cambios realizados y facilitando la gestión del historial del proyecto.

## **Desventajas de usar Markdown con GitHub** 

##### **Capacidades reducidas**

 Debido a que Markdown está pensado para ser un lenguaje simple, resulta limitado cuando se requiere incorporar elementos más complejos, como diseños avanzados, estructuras detalladas o componentes visuales más elaborados. En esos casos, es necesario complementarlo con HTML u otros lenguajes más completos.



##### **Poco práctico para documentación muy extensa**
 
 Cuando se trabaja con archivos de gran tamaño, como manuales amplios o textos muy largos, mantener la estructura y organización en Markdown puede volverse complicado. Gestionar capítulos, secciones y grandes volúmenes de contenido puede requerir herramientas más robustas.
